🗺 Thesis in One Page

The Core Problem

Credit risk models use black-box ensembles (RF, XGB, LGB) because they outperform simple models. But ECOA, GDPR, and the EU AI Act require transparent explanations. SHAP is the standard XAI tool but becomes unstable under class imbalance (which is always present in credit data). Can coalition-aware game-theoretic methods do better?

What Was Done

Datasets
5 credit datasets spanning 1%–30% default rates (Coursera, LC-4%, LC-10%, Taiwan, German)
420 total configs = 3 models × 4 samplers × 7 methods × 5 datasets
Models
Random Forest, XGBoost, LightGBM — evaluated under 4-fold stratified CV
Tuned via RandomizedSearchCV (15 iterations) on held-out tuning pool
Samplers
None, SMOTE, SMOTETomek, Cost-Sensitive learning
Applied inside cross-val folds (no leakage) via ImbPipeline
Methods
SHAP, Banzhaf, Myerson, Owen-Domain, Owen-Data, Owen-Model, R-Myerson
All applied on stratified test sample (n=30, min 5 defaults) per fold

Main Findings

R-Myerson wins: Perfect Kuncheva (K_c = 1.000) on ALL 5 datasets. Highest S(α=0.5) on most datasets. Best regulatory consistency (Jaccard = 1.000 vs SHAP range 0.169–0.643).
SMOTE paradox: SMOTE improves the mathematical I score (more stable attributions) but degrades AUC-ROC. This is because it creates synthetic data — explanations are stable but based on a fake distribution, not real feature-label relationships.
AUC–I trade-off exists, but only under moderate-to-severe imbalance: Significant negative Spearman ρ on Taiwan (−0.258), LC-10% (−0.395), Coursera (−0.391). No significant correlation on German or LC-4%.

🔑 The Characteristic Function — The Foundation of Everything

Your supervisor asked: "How is the characteristic function implemented?"
Your answer "it is averaged" was partially right. Here is the complete, correct answer.

Theory: What is v(S)?

In cooperative game theory, the characteristic function assigns a "worth" to each coalition (subset) of players. In machine learning:

v(S) = f(x_S)

= the model's output when ONLY the features in coalition S are available.
Features NOT in S must be "marginalized out" — integrated over their distribution.

The Critical Question: How is "marginalization" done?

There are two main approaches. TreeSHAP uses the interventional (conditional) approach for speed, but the exact mechanism matters:

ApproachHow v(S) is computedWhat it means
Interventional (SHAP default)Features not in S are replaced with values from a background/reference dataset. Average over background samples.Evaluates model on possibly out-of-distribution input combinations
ConditionalFeatures not in S are integrated out conditional on S's values — their actual joint distributionMore realistic but computationally expensive
TreeSHAP path-dependentFollows the decision tree path; at each split, if a feature is NOT in S, averages left/right outcomes weighted by training data proportionsExact, polynomial-time, no background needed — but path-dependent variant can give different answers for same function expressed differently

In Your Code: How is v(S) actually computed?

For SHAP (TreeSHAP):

def get_shap_values(pipe, X_test):
    clf = pipe.named_steps['clf']
    X_proc = pipe.named_steps['prep'].transform(X_test)
    
    explainer = shap.TreeExplainer(clf)
    sv = explainer.shap_values(X_proc)
    
    if isinstance(sv, list): return sv[1]   # binary: take class 1
    elif sv.ndim == 3: return sv[:, :, 1]   # 3D output: take class 1
    return sv
Key Answer for Supervisor: TreeSHAP computes v(S) internally within the decision tree structure. For each node where a feature is NOT in S, it averages the model predictions over the training samples that pass through that node — this is the "conditional expectation given the tree path." It does NOT simply average across all background samples. The SHAP values you get back are the Shapley values of this game, satisfying the efficiency axiom: sum of all φ_i = f(x) − baseline.

For Banzhaf (your code uses Monte Carlo):

def compute_banzhaf(pipe, X_test, n_samples=2, max_instances=2):
    # For each instance x and feature f:
    for _ in range(n_samples):
        coal = np.random.binomial(1, 0.5, n_feat)  # random coalition
        x_with = x.copy()
        x_with[0, f] = x[0, f] * coal[f]           # feature f present if coal[f]=1
        p1 = clf.predict_proba(x_with)[0, 1]
        
        x_without = x.copy()
        x_without[0, f] = 0                          # feature f absent = set to 0
        p0 = clf.predict_proba(x_without)[0, 1]
        
        contrib.append(p1 - p0)
    mat[i, f] = np.mean(contrib)
Important limitation: Setting absent features to 0 (instead of sampling from background) is a simplification. This uses zero as the "null" state, which may not represent the true marginal expectation. Also, n_samples=2, max_instances=2 is extremely conservative due to the 1,680-configuration experimental grid. This is the main reason Banzhaf ranked last — approximation budget, not fundamental method weakness.

1. SHAP — The Baseline

Theory

φ_i^SHAP = Σ_{S ⊆ N\{i}} [|S|!(d-|S|-1)!/d!] × [v(S∪{i}) − v(S)]

= Expected marginal contribution of feature i, averaged over all possible orderings of features

Weight = |S|!(d-|S|-1)!/d! ensures each ordering is equally likely

The 4 Axioms (know these!)

AxiomWhat it meansWhy it matters for credit
EfficiencyΣφ_i = f(x) − baselineAttributions sum to the actual prediction; full accountability
SymmetryIf two features contribute identically in all coalitions, they get equal attributionFairness between correlated features
DummyIf a feature contributes nothing to any coalition, φ_i = 0Irrelevant features get zero attribution
Additivityφ(v+w) = φ(v) + φ(w)Explanations for combined models combine linearly

Weaknesses (motivating your thesis)

  • Feature independence assumption: Standard SHAP treats features as independent when computing v(S). This causes out-of-distribution evaluations (features combined in ways never seen in training).
  • Instability under imbalance: Chen (2024) and Ballegeer (2025) show SHAP rankings change significantly as imbalance increases, measured by CV and SRA.
  • No structural awareness: Treats all features as independent players; ignores natural groupings (e.g., all repayment-history features).
  • Implementation non-invariance: Path-dependent TreeSHAP can produce different rankings for two tree structures computing the same function (Filom 2024).

Your Result

SHAP's cross-fold Jaccard ranged from 0.169 to 0.643 — failing the 0.80 regulatory threshold on ALL 5 datasets.

2. Banzhaf Value

Theory

φ_i^Banzhaf = (1/2^{d-1}) × Σ_{S ⊆ N\{i}} [v(S∪{i}) − v(S)]

Key difference from Shapley: ALL coalitions get EQUAL weight (1/2^{d-1})
Shapley weights smaller and larger coalitions more; Banzhaf weights all equally

NO efficiency axiom: Σφ_i ≠ f(x) in general

Theoretical Robustness Argument (Wang 2023)

Banzhaf has the minimal Lipschitz constant among all semivalues. This means small perturbations in v(S) estimates produce smaller changes in attributions than any other semivalue including Shapley. This is WHY it should be theoretically more stable.

Why it FAILED in your experiments

The approximation budget was too conservative: n_samples=2, max_instances=2 per configuration in the experimental grid.

This means for each feature, only 2 random coalitions were sampled, and only on 2 test instances. With such a tiny budget, the approximation variance completely overwhelms the theoretical stability advantage. The result: artificially high magnitude stability (uniform, low-variance outputs near zero) but terrible directional consistency (cosine similarity as low as 0.216 on LC-10%).

3. Myerson Value

Theory

Myerson (1977): In some games, not all coalitions are feasible — players can only cooperate if they're "connected" via a communication graph G. Only coalitions forming a connected subgraph of G are admissible.

φ_i^Myerson = φ_i(v^G)

where v^G(S) = Σ_{T ∈ connected_components(S in G)} v(T)

= The Shapley value of the RESTRICTED game, where disconnected coalitions are broken into their connected components

Key insight: This prevents attributions from averaging over feature combinations that never co-occur in reality — the primary source of out-of-distribution bias in SHAP.

Your Approximation (Eq. 2.20)

φ_i^Myerson ≈ (1−β)φ_i^SHAP + β × (1/|N(i)|) × Σ_{j∈N(i)} φ_j^SHAP

β = 0.5 (neutral midpoint between pure SHAP and full neighbor averaging)
N(i) = neighbors of feature i in the communication graph
Graph built using Pearson correlation: edge exists if |ρ_ij| ≥ 0.25
This is a graph-smoothed heuristic of the exact Myerson value. It's adopted for computational tractability across 1,680 configurations. The exact computation would require re-running the game for every connected component separately.

Graph Construction in Code

# Built per fold using TRAINING DATA ONLY (no test leakage)

Smoothing Computation

def compute_myerson(pipe, X_test, G, alpha=0.5):  # alpha = β in thesis
    shap_vals = get_shap_values(pipe, X_test)
    shap_mean = shap_vals.mean(axis=0)  # mean SHAP across test instances
    
    for i in range(d):
        neighbors = list(G.neighbors(i))
        if len(neighbors) == 0:
            phi[i] = shap_mean[i]  # isolated node: keep SHAP
        else:
            neighbor_mean = mean(shap_mean[j] for j in neighbors)
            phi[i] = (1 - alpha) * shap_mean[i] + alpha * neighbor_mean
    
    # Return tiled — all instances get the SAME Myerson vector
    return np.tile(phi, (n_tile, 1)), shap_vals, shap_mean
Note: The Myerson smoothing is applied to the mean SHAP vector across test instances, then tiled back. All instances in a fold get the same Myerson attribution. This is why it lacks instance-level discrimination but achieves fold-level stability.

4. Owen Value Variants

Theory

Owen Value: Two-level Shapley decomposition

1. First: compute how much each GROUP contributes (inter-group Shapley game)
2. Then: within each group, distribute the group's share using another Shapley game

Key property: Efficiency holds at BOTH levels
When each group has 1 member → reduces to standard Shapley value

Your Implementation (Owen-inspired redistribution)

def owen_from_phi(phi, groups):
    # phi = mean TreeSHAP vector (d-dimensional)
    # groups = dict of group_name → [feature indices]
    
    for group_name, feats in groups.items():
        total = np.sum(np.abs(phi[feats]))   # total absolute contribution of group
        per = total / len(feats)              # equal share per member
        
        for f in feats:
            out[f] = sign(phi[f]) * per       # preserve sign, equalize magnitude

# Then tile for all instances:
owen_dom = np.tile(owen_from_phi(phi_base, domain_groups), (len(X_sample), 1))

Why 5 groups? (Important for defense)

Group Quality (Q)

Q = W / B

W = mean within-group Pearson correlation (features in same group correlate with each other)
B = mean between-group Pearson correlation (features across groups are less correlated)

Q > 1: groups are cohesive (within > between) → good grouping
Q < 1: groups are poorly defined 
def compute_Q(groups, dep):
    # dep = absolute correlation matrix
    within, across = [], []
    for i in range(d):
        for j in range(i+1, d):
            if group_id[i] == group_id[j]:
                within.append(dep[i, j])
            else:
                across.append(dep[i, j])
    return mean(within) / mean(across)

5. R-Myerson — Your Novel Contribution

The Big Idea

Myerson stabilises attributions by restricting coalitions to connected graph nodes. But it can still be variable because the smoothed SHAP values themselves vary across folds. R-Myerson adds a second layer: egalitarian redistribution of the Myerson vector's total magnitude.

φ_i^R-Myerson = sign(φ_i^Myerson) × (1/d) × Σ_j |φ_j^Myerson|

= preserve the DIRECTION of each feature's Myerson attribution
+ give ALL features the SAME magnitude (average of all Myerson magnitudes)

This is why K_c = 1.000: since all features have identical magnitudes,
the top-k features are determined solely by their SIGNS (positive = risk-increasing)
Signs are stable across folds → perfect Kuncheva.

Code

def adaptive_owen_myerson(phi_my):
    # phi_my = mean Myerson vector (d-dimensional, from compute_myerson)
    d_local = len(phi_my)
    total = np.sum(np.abs(phi_my))       # total magnitude
    per = total / d_local                 # equal share for all features
    
    return np.array([
        (np.sign(x) if x != 0 else 1) * per
        for x in phi_my
    ])

# Applied:
r_myerson = np.tile(
    adaptive_owen_myerson(m_vals.mean(axis=0)),  # m_vals from compute_myerson
    (len(X_sample), 1)
)

Theoretical Motivation

InspirationMechanismHow R-Myerson uses it
Myerson (1977)Graph-restricted coalitions eliminate unrealistic combinationsStep 1: compute φ^Myerson as graph-smoothed SHAP
Dutta & Ray (1989)Egalitarian solutions are Lorenz-maximal; equal redistribution when stability > differentiationStep 2: redistribute magnitudes uniformly across all d features
Wang (2023) — BanzhafUniform coalition weighting → minimal Lipschitz constant → stabilityAnalogy: uniform MAGNITUDE redistribution → stability of ranking (by sign)
Abe (2023)In-group egalitarian Owen: uniform redistribution within groupsR-Myerson applies this globally (whole feature set = one "group")

The Trade-off R-Myerson Makes

R-Myerson sacrifices magnitude discrimination for sign stability.
All features get the same absolute attribution. You CANNOT use R-Myerson to rank denial reasons by magnitude (feature A contributed 0.05 more than B). You CAN use it to identify which features are risk-increasing (+) vs risk-decreasing (−) consistently.

For adverse action notices: R-Myerson identifies the consistent SET of risk factors. For RANKED denial reasons, pair R-Myerson with SHAP or Owen-Domain for magnitude ranking.

Why K_c = 1.000 is "by construction"

Since all d features get identical magnitude per fold, the top-k feature set is determined solely by which features have positive signs. If signs are stable across folds (which they are, since Myerson smoothing itself is stable), then the same k features always appear in the top-k. Hence perfect Kuncheva. This is a structural property, not a statistical coincidence.

📐 Interpretability Metrics — Code and Formulas

1. Magnitude Stability (S_var)

CV_j = σ_j / (μ_j + ε)    where σ_j = std of |φ_{fold,j}|, μ_j = mean
S_var = 1 − mean(CV across features) 
def stability_cv(expl_list):
    # expl_list: list of attribution arrays, one per fold
    arr = np.stack([np.abs(e) for e in expl_list])  # stack folds
    mean = arr.mean(axis=0) + 1e-8                    # mean per feature
    return float(np.mean(arr.std(axis=0) / mean))    # mean CV across features
# Note: code returns CV (instability). Thesis uses S_var = 1 - CV

2. Kuncheva Index (K_c)

K_{p,q} = (|A_p ∩ A_q| − k²/d) / (k − k²/d)

= (actual overlap − chance overlap) / (maximum overlap − chance overlap)

K_c = average over all fold pairs
Range: [−1, 1]. K_c = 1: perfect stability. K_c = 0: random (no better than chance). 
def kuncheva_index(expl_list, k=10):
    sets = []
    for exp in expl_list:
        imp = np.abs(exp).mean(axis=0)           # mean attribution magnitude
        top_k = set(np.argsort(imp)[-k:])        # top-k feature indices
        sets.append(top_k)
    
    ki_vals = []
    for i in range(len(sets)):
        for j in range(i+1, len(sets)):
            overlap = len(sets[i] & sets[j])      # intersection size
            chance = k**2 / d                     # expected random overlap
            ki = (overlap - chance) / (k - chance) if k != d else 0.0
            ki_vals.append(ki)
    return np.mean(ki_vals)
Why not Jaccard? Jaccard = |A∩B|/|A∪B| doesn't correct for chance. If you pick k=5 features from d=100, there's always some random overlap. Kuncheva subtracts this chance baseline, making it a genuine measure of above-chance consistency.

3. Cosine Directional Consistency (C)

C_{p,q} = (φ̄_p · φ̄_q) / (||φ̄_p|| × ||φ̄_q||)

where φ̄_fold = mean attribution vector across test instances in that fold
C = average cosine similarity over all fold pairs 
def cosine_directional_consistency(expl_list):
    mean_vecs = [np.mean(e, axis=0).ravel() for e in expl_list]  # mean per fold
    sims = [1 - cosine(mean_vecs[p], mean_vecs[s])
            for p in range(len(mean_vecs)) 
            for s in range(p+1, len(mean_vecs))]
    return np.mean(sims)

4. Composite Interpretability Score (I)

I = (1/3) × S_var + (1/3) × K_c + (1/3) × C

Equal weights because: no consensus in literature on relative importance (Pawlicki 2024).
In practice: weights can be adjusted for regulatory context (e.g., higher K_c weight for adverse action notices). 
def interpretability_score(cv_val, kun_val, cos_val, beta=1/3):
    return beta * (1 - cv_val) + beta * kun_val + beta * cos_val
# Note: (1 - cv_val) = S_var

5. Performance-Interpretability Trade-off (S(α))

z̃ = (z − z_min) / (z_max − z_min) [min-max normalization per column]

S(α) = α × AUC̃ + (1−α) × Ĩ

α = 0.5: equal weight to accuracy and interpretability (baseline)
α = 1: purely predictive; α = 0: purely interpretable 
def normalize(s):
    return (s - s.min()) / (s.max() - s.min() + 1e-8)

def performance_interpretability_score(auc_series, I_series, alpha=0.5):
    return alpha * normalize(auc_series) + (1 - alpha) * normalize(I_series)

# Applied AFTER all configurations are collected:
metrics['S(α=0.5)'] = performance_interpretability_score(metrics['AUC'], metrics['I'])
Important: S(α) is computed by normalizing across ALL configurations in one dataset (not globally). This means the S score is relative within a dataset. The global ranking uses the score from within each dataset.

🔥 Tricky Q&A — Expected Defense Questions

Q: You told your supervisor the characteristic function is "averaged" — was that correct?
Partially correct, but incomplete. For TreeSHAP: the characteristic function v(S) is computed internally by TreeSHAP by following the decision tree structure. For nodes where a feature in S is used for splitting, the path is followed normally. For nodes where a feature NOT in S is encountered, the prediction is averaged over both child paths, weighted by the proportion of training samples that went left vs right. This is a conditional expectation given the tree path — not a simple average over all background samples.

For your Owen/Myerson variants: you don't recompute v(S) at all. You take the pre-computed TreeSHAP φ values and redistribute them. So the characteristic function is "inherited" from TreeSHAP and the redistribution happens at the attribution level, not the value function level.
Q: Your R-Myerson gets perfect K_c = 1.000 by construction. Isn't this "cheating"?
No — and you should address this directly and honestly. K_c = 1.000 is a structural property of the method, explicitly acknowledged in the thesis. Because all features get equal magnitude, the top-k set is determined purely by sign. If signs are stable across folds (which they are due to Myerson's graph-smoothing), the ranking is perfectly consistent.

This is not cheating because: (1) the method transparently achieves this through its design; (2) sign stability is itself a meaningful and desirable regulatory property — adverse action notices require knowing which features are risk-increasing, not their exact magnitude ranking; (3) the trade-off is openly acknowledged: R-Myerson sacrifices magnitude discrimination in exchange for sign consistency. For ranked denial reasons, the thesis explicitly recommends pairing R-Myerson with SHAP or Owen attributions.
Q: Why did you use Pearson correlation for graph construction and Owen grouping, not Spearman?
Methodological consistency with the linear feature space. After one-hot encoding and StandardScaler, the feature space is fully numeric and approximately linear. SHAP values themselves behave linearly in expectation. Under these conditions, Pearson correlation provides a geometrically coherent similarity measure aligned with the Euclidean structure of the agglomerative clustering distance metric.

Non-parametric alternatives (Spearman, Kendall, Jaccard for binary features) are valid — the thesis explicitly acknowledges this. The choice is a deliberate methodological decision for consistency, not a claim that Pearson is universally superior. Future work should test sensitivity to correlation measure choice.
Q: Is your Myerson implementation actually the Myerson value?
No — and you say so explicitly in the thesis. The exact Myerson value (Eq. 2.17) requires computing the Shapley value of the restricted game v^G, where coalitions are restricted to connected subgraphs. This would require re-running game computations for every connected component of every coalition — exponential complexity.

Your implementation (Eq. 2.20) is a graph-smoothed heuristic: φ_i^Myerson ≈ (1−β)φ_i^SHAP + β × average of neighbor SHAP values. This is described as "a graph-smoothed heuristic approximation... translating the principles of graph-restricted games into a scalable machine learning regulariser." It's adopted for computational tractability across 1,680 configurations.
Q: SMOTE improved the I score but you recommend against it. Isn't that contradictory?
This is the core tension of the thesis and you should address it confidently. The I score improvement from SMOTE is artificial. SMOTE creates synthetic minority instances by interpolating between real ones. When models are trained on this synthetic data, decision boundaries become smoother and more regular — which makes SHAP attributions more consistent across folds (lower CV, higher Kuncheva).

However, this stability is based on a synthetic data distribution, not the true feature-label relationships of real defaulters. As Cemernek (2024) shows, this jeopardises model understandability: explanations are stable descriptions of a synthetic world, not the real credit risk landscape.

The S(α) framework is designed precisely to expose this paradox: SMOTE gets high I but low AUC, giving it a lower S(α) than cost-sensitive approaches that achieve both good AUC and reasonable (genuinely earned) stability.
Q: Why is there no AUC–I trade-off in German (30%) and LC-4%?
Two very different reasons for two very different datasets:

German (30%): Near-balanced class distribution provides enough minority examples that any model configuration can learn good decision boundaries AND produce stable attributions. With 30% defaults in 1,000 samples (300 default cases), the minority class is abundant enough for joint optimization. Hence ρ = 0.027, p = 0.809 — no trade-off exists.

LC-4%: The opposite reason. Only 66 actual default events in the evaluation pool (after 50/50 tuning split, ~824 evaluation samples, ~33 defaults). Both AUC and I have high estimation noise from such few minority examples. There isn't enough statistical signal to detect the latent trade-off even if it exists. Hence ρ = −0.017, p = 0.877 — statistically uninformative, not structurally absent.
Q: How do you know the Q vs I negative correlation (German, LC-4%) is a redistribution artifact and not a genuine finding?
In German: High Q scores arise when agglomerative clustering groups one-hot encoded dummy variables together (e.g., all levels of "checking_account_status" end up in one cluster, making within-group correlation near-perfect). But this grouping lacks semantic coherence — redistributing magnitude uniformly within a cluster of OHE dummies produces unstable attributions because the individual dummies don't carry independent information.

In LC-4%: 24 of 36 Owen-Data/Owen-Model configurations collapsed ALL features into a single cluster (Q = 1.0, meaning no between-group correlation could be computed). When this happens, the entire feature set gets treated as one "group," and Owen redistribution equalizes ALL magnitudes — producing the same outcome as R-Myerson. This artificially spikes I while Q=1.0 is a degeneracy signal, not a genuine quality measure. The negative relationship arises from this edge case, not a theoretically meaningful pattern.
Q: Why τ = 0.25 for the Myerson graph threshold?
This was chosen as a pragmatic balance. A very low threshold (e.g., τ = 0.10) would create a nearly fully connected graph — almost every feature pair gets an edge, neighbor averaging would reduce all attributions to nearly identical values (equivalent to full egalitarian redistribution), eliminating structural information. A very high threshold (e.g., τ = 0.70) would create a sparse graph with many isolated nodes — most features would get pure SHAP values with no graph smoothing, making Myerson nearly identical to SHAP.

τ = 0.25 captures meaningful statistical dependencies while avoiding overly dense graphs. Sensitivity analysis on τ is explicitly identified as future work in Section 5.5. This is an honest acknowledgment that the choice is not rigorously optimized.
Q: What exactly is the "efficiency axiom" and why does it matter for adverse action notices?
The efficiency axiom states: Σ φ_i = f(x) − E[f(x)] (or f(x) − baseline).

In plain language: the sum of all feature attributions equals the model's prediction minus the baseline prediction (e.g., average prediction across the training set). This means the explanation is complete — it fully accounts for why the model made this particular prediction relative to baseline.

For adverse action notices: efficiency means you can say "your denial probability was 0.65. The baseline rate is 0.30. The 0.35 excess is explained by: +0.20 from poor repayment history, +0.10 from high DTI, +0.05 from short credit history..." — fully auditable and transparent. Without efficiency, the attributions might leave unexplained residuals, violating ECOA's requirement for specific, complete explanations.

Note: R-Myerson's egalitarian redistribution does NOT satisfy efficiency in the traditional Shapley sense, because magnitudes are equalized. R-Myerson's contribution is directional consistency, not efficiency-based explanation completeness.
Q: Why is Banzhaf not efficient (and what are the implications)?
Banzhaf values don't satisfy Σ φ_i = f(x) − baseline in general. They measure the probability that a feature is pivotal in a random coalition, weighted uniformly. Unlike Shapley, where the combinatorial weights sum to 1 ensuring full allocation, Banzhaf's uniform 1/2^{d-1} weights don't partition the total prediction neatly.

Implication for adverse action: You cannot use Banzhaf attributions in an additive explanation that sums to the prediction — the numbers won't "add up" to the denial probability. This makes it harder to present in a CFPB-compliant adverse action notice that must explain the specific degree of each factor's contribution. SHAP (and technically Owen) are preferred for their efficiency property in regulatory contexts.
Q: What is the Rashomon effect and how does it relate to your thesis?
The Rashomon effect (named after Kurosawa's film about multiple conflicting accounts of the same event) in machine learning: for any given prediction error level, there exists a large set of diverse models that achieve equally optimal accuracy. Implication: at least one member of this set is likely to be simpler and more interpretable.

Relation to your thesis: It undermines the common assumption that accuracy–interpretability trade-off is inevitable. If you could search the full model space (rather than just gradient boosting), you'd likely find models that are both accurate AND interpretable. The trade-off persists mainly because modern ML tools (like XGBoost) are designed to minimize prediction error and adopt the first complex solution found, not search for the interpretable one (Majumdar 2025).

Your thesis finds this empirically: in German data (balanced), there IS no trade-off (ρ ≈ 0). Only under severe imbalance does the trade-off emerge — which supports the regime-dependent view of the Rashomon effect.
Q: Why did you choose AUC-ROC over PR-AUC or F1 for imbalanced datasets?
AUC-ROC is threshold-independent and class-prevalence invariant — it measures ranking quality across the full probability distribution, not at a specific cutoff. This makes it suitable for comparing models across 5 datasets with default rates ranging from 1% to 30%: a PR-AUC of 0.5 means something very different on a 1% dataset vs a 30% dataset, making cross-dataset comparison invalid.

However, AUC-ROC has a known limitation in credit: a model can have high AUC-ROC but assign all applicants probabilities below the operational denial threshold (which your analysis confirmed — 100% miss rate on Coursera and LC-10% for no-resampling models). This is precisely why the thesis conducts the credit decision bias analysis (Section 4.1.8) — to demonstrate that AUC-ROC alone is insufficient for deployment and cost-sensitive learning is necessary.
Q: What is Cliff's delta and why use it alongside Wilcoxon?
Cliff's delta is a non-parametric effect size measure: δ = (# pairs where x > y − # pairs where x < y) / total pairs. Range [−1, +1]. Interpretation: |δ| < 0.147 negligible; < 0.33 small; < 0.474 medium; ≥ 0.474 large.

Wilcoxon signed-rank test tells you IF a difference is statistically significant (p < 0.05). Cliff's delta tells you HOW LARGE that difference is in practice. This matters because with large samples (your 12 model-sampler configurations per dataset), even tiny, practically irrelevant differences can become statistically significant. By reporting both, you show that R-Myerson not only outperforms other methods significantly (Wilcoxon p < 0.05) but also by a large effect (|δ| ≥ 0.474 in most comparisons).
Q: Your thesis says R-Myerson is the "first" to combine Myerson + egalitarian redistribution. How can you be sure?
You should be careful but confident here. The thesis states: "to the best of the author's knowledge" — this qualifier is important and should be included in every defense answer about novelty. The literature review used SALSA methodology, searched across Scopus, Web of Science, IEEE Xplore, ScienceDirect, SpringerLink, arXiv, and Semantic Scholar. No paper was found that: (1) constructs a Myerson communication graph from empirical feature correlations in tabular credit data, AND (2) applies egalitarian redistribution to the resulting Myerson attributions for ML feature attribution.

Boruah (2025) uses k-Egalitarian Myerson for cooperative network games (social allocation, NOT supervised learning). Abe (2023) applies in-group egalitarian redistribution to Owen values, not Myerson. The bibliometric analysis (Litmaps figure) confirms sparse cross-pollination between cooperative game theory and XAI literature. The gap is real; the claim is defensible with the "to the best of our knowledge" qualifier.
Q: What is the Kuncheva correction for chance, and why does it matter?
If you randomly select k features from d total features, in two independent selections the expected overlap by chance is k²/d. For example, if k=10 and d=100: expected random overlap = 100/100 = 1 feature. So if two methods overlap on 3 features, Jaccard would say 3/17 ≈ 0.18. But Kuncheva corrects: (3 − 1)/(10 − 1) ≈ 0.22 — still low, but now appropriately compared to a random baseline.

This matters when k << d. In your datasets with d=82 features (Taiwan after OHE) and k=10, chance overlap is 100/82 ≈ 1.2 features. Without correction, any method overlapping on 2 features might look "stable" with Jaccard, but after Kuncheva correction you see it's barely above chance. The correction prevents misleadingly optimistic stability assessments.
Q: Could you have used a different coalition structure for Myerson than Pearson correlation?
Yes, and this is a valid criticism to address proactively. Alternative options include: mutual information (captures non-linear dependencies), Spearman rank correlation (monotonic, robust to outliers), partial correlation (controls for confounders), or domain-expert defined connectivity. For binary OHE features specifically, Jaccard distance is recommended by DavilaPena (2025).

Pearson was chosen for methodological consistency — the same measure used in Owen-Data clustering and the Myerson graph ensures the framework is internally coherent. After OHE and StandardScaler, the feature space is sufficiently linear for Pearson to be appropriate. The τ = 0.25 threshold further filters out weak, potentially spurious correlations. The effect of alternative graph construction methods is identified as a direction for future research.
Q: How do you ensure your Banzhaf implementation is even approximately correct with n_samples=2?
You should be honest about this limitation. With n_samples=2 coalitions sampled per feature per instance, and only max_instances=2 instances evaluated, the Banzhaf approximation is extremely noisy. Wang (2023) establishes that the approximation variance can overwhelm the theoretical stability advantage when the budget is too tight.

The thesis explicitly acknowledges this in Section 5.4 (Limitations) and in the Computational Sampling Parameters section: "Banzhaf used a conservative Monte Carlo budget... constrained by the computational demands of the full experimental grid (1,680 configurations). Its implications for Banzhaf's observed instability are discussed in Chapter 5." The result is: Banzhaf's poor performance reflects approximation budget limitations, not a fundamental weakness of the Banzhaf value. Rerunning with n_samples=500+ would likely improve Banzhaf substantially — this is identified as a future research direction.
